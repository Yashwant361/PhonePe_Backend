# Security Policy

Thank you for helping improve the security of this project.

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.x     | Yes       |
| <1.0    | No        |

## Reporting a Vulnerability

Please do not open a public issue for security vulnerabilities.

Instead, report privately by contacting the maintainer through GitHub profile or email.

Include:

- Description of the issue
- Steps to reproduce
- Possible impact
- Suggested fix (optional)

## Response Time

We aim to review reports within 7 days.

## Scope

Examples:

- Authentication bypass
- JWT issues
- MPIN validation flaws
- Transaction manipulation
- Sensitive data exposure
